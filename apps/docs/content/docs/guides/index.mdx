Guides are organized by what you are trying to build, not by package name. Use them when you need a path through multiple packages.

| Goal | Read |
| --- | --- |
| Build the first module/controller/service slice | [First Feature](./first-feature) |
| Choose packages for a task | [Package Selection](./package-selection) |
| Build REST APIs | [HTTP API](./http-api) |
| Move across runtime adapters | [Runtime Adapters](./runtime-adapters) |
| Validate inputs and shape outputs | [Validation & Serialization](./validation-serialization) |
| Add bearer or strategy-based auth | [Auth](./auth) |
| Add databases, Redis, cache, and queues | [Persistence](./persistence) |
| Add WebSockets or Socket.IO | [Realtime](./realtime) |
| Add events, CQRS, jobs, cron, and notifications | [Messaging & Workflows](./messaging-workflows) |
| Add metrics, health, throttling, and diagnostics | [Operations](./operations) |
| Test applications and packages | [Testing](./testing) |
| Use the CLI and generators | [CLI](./cli) |

Each guide links to the package reference pages that own the underlying contracts.
