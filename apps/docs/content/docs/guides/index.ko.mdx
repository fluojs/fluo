가이드는 패키지 이름이 아니라 만들려는 결과를 기준으로 구성됩니다. 여러 패키지를 함께 사용해야 할 때 이 섹션에서 경로를 찾으세요.

| 목표 | 읽을 문서 |
| --- | --- |
| 첫 module/controller/service slice 만들기 | [첫 기능 만들기](./first-feature) |
| 작업에 맞는 패키지 선택 | [패키지 선택](./package-selection) |
| REST API 만들기 | [HTTP API](./http-api) |
| 런타임 어댑터 이동 | [런타임 어댑터](./runtime-adapters) |
| 입력 검증과 출력 형태 구성 | [검증과 직렬화](./validation-serialization) |
| bearer 또는 strategy 기반 인증 추가 | [인증](./auth) |
| 데이터베이스, Redis, cache, queue 추가 | [영속성](./persistence) |
| WebSocket 또는 Socket.IO 추가 | [실시간](./realtime) |
| event, CQRS, job, cron, notification 추가 | [메시징과 워크플로](./messaging-workflows) |
| metrics, health, throttling, diagnostics 추가 | [운영](./operations) |
| 애플리케이션과 패키지 테스트 | [테스트](./testing) |
| CLI와 generator 사용 | [CLI](./cli) |

각 가이드는 하위 계약을 소유하는 패키지 레퍼런스로 연결됩니다.
