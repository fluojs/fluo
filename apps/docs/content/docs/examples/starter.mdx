The repository's `examples/` directory is the executable companion to these docs. Start with the smallest starter, then move to feature-specific examples as your application needs grow.

## Recommended reading order

1. Create an app with the CLI from [Quick Start](../getting-started/quick-start).
2. Compare the generated structure with `examples/README.md`.
3. Add one package family at a time, following [Package Selection](../guides/package-selection).

## Documentation contract

Examples should prove documented behavior. If a guide promises a runtime capability, the matching example or test should exercise that capability instead of relying on prose alone.
