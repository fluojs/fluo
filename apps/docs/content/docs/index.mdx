fluo is a standard-first TypeScript backend framework built on TC39 standard decorators, explicit dependency boundaries, and metadata-free runtime wiring.

## Why fluo?

Most decorator-heavy TypeScript frameworks depend on legacy compiler behavior. fluo keeps the programming model close to the JavaScript language path: no `experimentalDecorators`, no `emitDecoratorMetadata`, and no hidden reflection requirement.

## How these docs are organized

- **Getting Started** gives you the shortest working path, inspired by Hono's fast quick-start flow.
- **Concepts** explains the standard-decorator and explicit dependency model before you scale up.
- **Guides** provides task-oriented decisions for HTTP, auth, persistence, realtime, messaging, operations, testing, and tooling.
- **Package Reference** keeps all 40 official packages discoverable by family.
- **Examples** points to runnable starter, RealWorld, auth, and operations paths that prove the documented behavior.
- **Governance** explains how English and Korean documentation stay in sync.
