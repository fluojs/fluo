fluo는 TC39 표준 데코레이터, 명시적 의존성 경계, 메타데이터 없는 런타임 구성을 기반으로 하는 standard-first TypeScript 백엔드 프레임워크입니다.

## 왜 fluo인가요?

대부분의 데코레이터 중심 TypeScript 프레임워크는 legacy 컴파일러 동작에 의존합니다. fluo는 JavaScript 언어 경로에 가까운 프로그래밍 모델을 유지합니다. `experimentalDecorators`, `emitDecoratorMetadata`, 숨겨진 reflection 요구사항이 없습니다.

## 이 문서의 구성

- **시작하기**는 Hono의 빠른 quick-start 흐름처럼 가장 짧은 실행 경로를 제공합니다.
- **개념**은 확장 전에 표준 데코레이터와 명시적 의존성 모델을 설명합니다.
- **가이드**는 HTTP, 인증, 영속성, 실시간, 메시징, 운영, 테스트, 도구 흐름을 작업 중심으로 제공합니다.
- **패키지 레퍼런스**는 40개 공식 패키지를 family별로 찾을 수 있게 합니다.
- **예제**는 스타터, RealWorld, 인증, 운영 흐름처럼 문서화된 동작을 증명하는 실행 경로로 안내합니다.
- **거버넌스**는 영어와 한국어 문서가 어떻게 동기화되는지 설명합니다.
