---
title: Package Doc Template
description: The structure every package reference page should follow.
---

Use this shape when adding dedicated package pages beyond family overviews.

## Template

1. Overview
2. When to use
3. Installation
4. Basic usage
5. Integration points
6. API surface
7. Configuration
8. Runtime compatibility
9. Related packages
10. Stability and docs status

Package reference pages should avoid duplicating entire README files. They should route readers to the right package, guide, example, and governed source document.
