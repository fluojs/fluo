# Chapter 0. Welcome to fluo: The Standard-First Framework

**fluo**에 오신 것을 환영합니다. fluo는 웹 개발의 향후 10년을 위해 바닥부터 설계된 현대적인 TypeScript 백엔드 프레임워크입니다. 기존 프레임워크의 "마법"과 레거시 부채 없이 확장 가능하고 고성능이며 미래 지향적인 서버측 애플리케이션을 구축하는 방법을 찾고 있다면 제대로 찾아오셨습니다.

이 책은 여러분을 단순한 호기심을 가진 개발자에서 능숙한 fluo 아키텍트로 성장시키기 위해 설계되었습니다. 우리는 단순히 문법만 살펴보는 것이 아니라, **FluoBlog**라는 실제 세계에서 즉시 사용 가능한 프로덕션급 애플리케이션을 함께 구축할 것입니다. 이 과정에서 왜 표준이 중요한지, 유지보수성을 위해 명시적 설계(Explicit Design)를 어떻게 활용하는지, 그리고 Node.js부터 Edge까지 모든 런타임에 코드를 배포하는 방법을 배우게 될 것입니다.

## What is fluo?

코드로 들어가기 전에 fluo가 무엇인지 정의해 보겠습니다. 오늘날 대부분의 TypeScript 프레임워크는 수년 전에 제안되었지만 공식 JavaScript 언어의 일부가 되지 못한 실험적인 기능들에 의존합니다. fluo는 **표준 우선(Standard-First)**을 지향함으로써 이 고리를 끊습니다.

fluo는 완전히 **TC39 Stage 3 Decorator** 사양을 기반으로 구축되었습니다. 이는 fluo가 컴파일러 트릭이 아니라, 실제 JavaScript 런타임의 일부가 될 공식 기능을 사용한다는 것을 의미합니다. 그 결과로 탄생한 프레임워크는 다음과 같은 특징을 가집니다.

- **Lean**: 무거운 reflection 라이브러리나 숨겨진 메타데이터 부풀리기가 없습니다.
- **Fast**: 빠른 시작 시간(Cold Start)과 낮은 메모리 사용량을 자랑합니다.
- **Explicit**: 의존성이 어떻게 연결되어 있는지 정확히 확인할 수 있습니다.
- **Portable**: 동일한 코드가 Node.js, Bun, Deno, 그리고 Cloudflare Workers에서 실행됩니다.

## Why This Book?

fluo에 대한 문서는 이미 많이 존재하지만, 공식 문서는 종종 특정 기능이 "어떻게" 작동하는지에 집중합니다. 이 책은 **"왜(Why)"**와 **"흐름(Flow)"**에 집중합니다.

새로운 프레임워크를 배우는 것이 부담스러울 수 있다는 점을 잘 알고 있습니다. 그래서 이 책은 누적된 경로를 따릅니다. 환경 설정과 같은 절대적인 기초부터 시작하여 라우팅, 데이터 영속성, 보안, 그리고 마지막으로 프로덕션 운영까지 복잡성의 층을 점진적으로 쌓아 나갑니다. 이 시리즈를 마칠 때쯤이면 여러분은 단순히 fluo를 사용하는 법을 아는 것이 아니라, 백엔드 시스템을 견고하게 만드는 아키텍처 패턴을 이해하게 될 것입니다.

## The FluoBlog Project

이 책의 핵심은 **FluoBlog**입니다. 단편적인 예제 대신, 우리는 21개 챕터에 걸쳐 완전한 블로그 엔진을 구축할 것입니다. 이것은 단순한 "Hello World" 프로젝트가 아닙니다. 우리는 다음과 같은 것들을 구현할 것입니다.

1. **A Modular Architecture**: 코드를 논리적 단위로 구성하기.
2. **RESTful APIs**: 복잡한 HTTP 요청과 응답 처리하기.
3. **Database Integration**: Prisma를 사용하여 영구 데이터 관리하기.
4. **JWT Authentication**: 업계 표준으로 API 보안 강화하기.
5. **Caching**: Redis로 성능 끌어올리기.
6. **Observability**: 프로덕션을 위한 헬스 체크와 메트릭 추가하기.

각 챕터는 FluoBlog에 새로운 기능을 추가하며, 이는 전문적인 백엔드 프로젝트의 실제 개발 라이프사이클을 반영합니다.

## Prerequisites

이 책을 최대한 활용하려면 다음이 필요합니다.

- **기초적인 JavaScript/TypeScript 지식**: 클래스, async/await, 기본적인 타입 선언에 익숙해야 합니다.
- **Node.js 설치**: fluo는 많은 런타임을 지원하지만, 우리는 Node.js와 `pnpm`을 기본 개발 환경으로 사용합니다.
- **터미널과 코드 에디터**: TypeScript 확장이 설치된 VS Code를 권장합니다.

NestJS, Express 또는 다른 백엔드 프레임워크에 대한 사전 경험은 **필요하지 않습니다**. 우리는 모든 개념을 기초부터 설명합니다.

## How to Read This Book

이 책은 5개의 논리적인 파트로 구성되어 있습니다.

### Part 0. Getting Started
fluo 뒤에 숨겨진 "왜"를 다루고, CLI를 설정하며, 핵심 빌딩 블록인 Module, Provider, Controller를 소개합니다. 또한 fluo의 비법 소스인 데코레이터의 신비를 벗겨냅니다.

### Part 1. Building the HTTP API
여기서는 애플리케이션의 "얼굴"을 만듭니다. 라우팅, 사용자 입력 처리(DTO), 데이터 검증 및 일관된 응답 반환에 대해 배웁니다. 또한 OpenAPI(Swagger)를 사용하여 API를 자동으로 문서화하는 방법도 다룹니다.

### Part 2. Configuration and Data
데이터베이스 없는 백엔드는 불완전합니다. 환경 기반 설정을 구축하고 Prisma를 사용하여 PostgreSQL 데이터베이스와 통신할 것입니다. 데이터 접근 패턴과 트랜잭션을 안전하게 처리하는 방법을 배우게 됩니다.

### Part 3. Authentication and Security
보안은 나중에 덧붙이는 것이 아닙니다. JSON Web Token(JWT) 인증을 구현하고, Passport를 사용한 전략 기반 권한 부여를 배우며, Rate Limiting을 통해 API를 남용으로부터 보호합니다.

### Part 4. Caching and Operations
마지막으로 FluoBlog를 실제 세계에 내놓을 준비를 합니다. 속도를 위한 Redis 캐싱을 추가하고, 모니터링을 위한 헬스 체크를 구현하며, 애플리케이션 상태를 추적하기 위해 Prometheus 메트릭을 설정합니다.

### Part 5. Testing and Completion
블로그에 버그가 없도록 단위 및 통합 테스트를 작성하는 것으로 마무리하고, 최종 프로덕션 체크리스트를 살펴봅니다.

## Orientation: The fluo Package Ecosystem

입문자들이 놀라는 것 중 하나는 fluo가 하나의 거대한 "블랙박스"가 아니라는 점입니다. fluo는 39개 이상의 전문화된 패키지 모음입니다. 이 입문 시리즈에서 우리는 주로 다음에 집중합니다.

### A Note on the "Standard-First" Approach
우리가 "표준 우선"이라고 말할 때, 그것은 개발자로서의 여러분의 커리어에 대한 약속이기도 합니다. fluo를 배움으로써 여러분은 공식 JavaScript Decorator API를 배우는 것입니다. 나중에 다른 도구나 다른 언어로 옮겨가더라도, 여기서 배우는 패턴들—의존성 주입, 모듈화, 명시적 설정—은 보편적으로 적용됩니다.

많은 개발자가 독자적인 DSL(Domain Specific Language)을 사용하는 프레임워크에 "갇혀 있다"고 느낍니다. fluo는 그 반대입니다. fluo는 여러분이 이미 알고 있는 언어의 확장입니다.

### Why Explicitness Matters
웹 개발 초기에는 "마법"이 기능으로 여겨졌습니다. 개발자가 무엇을 원하는지 추측할 수 있는 프레임워크가 인기를 끌었습니다. 하지만 애플리케이션이 거대한 마이크로서비스로 성장하면서 그 마법은 악몽이 되었습니다. 마법은 디버깅을 불가능하게 만들었고 리팩토링을 도박으로 만들었습니다.

fluo는 다른 길을 선택합니다. 우리는 **명시적인 것이 암시적인 것보다 낫다**고 믿습니다. fluo 컨트롤러를 보면 데이터가 어디서 오는지 정확히 알 수 있습니다. 모듈을 보면 그 모듈이 무엇을 제공하는지 정확히 알 수 있습니다. 처음에는 몇 줄의 코드가 더 필요할 수 있지만, 나중에 수백 시간의 디버깅 시간을 절약해 줍니다.
